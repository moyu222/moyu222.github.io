[!summary]

  • 分成两那个区域 Staging Area 和 Commits
  • Commit 中保存提交的版本 commit 0,1,2,…
  • Head 指向当前版本，master 指向一个分支的最新版本
  • 每一次commit新的commit首先复制前一个commit的内容然后根据staging area修改
  • git可以读取staging area 处的文件以判断是否别修改

git remote -v 命令用于查看与当前 Git 仓库关联的远程仓库的信息。这个命令会列出所有远程仓库的名称及其对应的 URL。
![[Pasted image 20241222220842.png]]
![[Pasted image 20241222220904.png]]
git push origin master 命令用于将本地的 master 分支的更改推送到名为 origin 的远程仓库

  • origin：这是默认的远程仓库名称，通常指向你克隆项目的源仓库。
  • master：这是你要推送的本地分支的名称。在许多项目中，master 是默认的主分支名称。
